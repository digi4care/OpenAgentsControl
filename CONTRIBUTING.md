## Quick Start

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create** a feature branch (`git checkout -b feature/my-feature`)
4. **Make** your changes
5. **Test** your changes (`./scripts/registry/validate-component.sh`)
6. **Commit** with conventional commits (`feat: add new agent`)
7. **Push** and create a pull request

## PR Guidelines

- Use **conventional commits** for PR titles
- Reference any related issues
- Include tests for new functionality
- Update documentation as needed

## Code Style

- **Markdown**: Clear, concise, with examples
- **TypeScript**: JSDoc comments, explicit types
- **Bash**: `set -e`, meaningful variable names
